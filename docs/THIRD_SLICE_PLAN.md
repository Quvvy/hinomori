# Third Slice Plan - Arcade Social Stage Stub

## Recommendation

Build the arcade as a **social stage stub**, not a rhythm game.

This slice proves queueing, watching, starting solo/1v1 stub matches, server-owned scoring, and rewards feeding back into the locker loop. It intentionally avoids rhythm charts, music assets, timing input, animations, and final arcade art.

## Scope

Status target: Prototype

In scope:

- one placeholder cabinet pair
- ProximityPrompt-driven arcade menu
- queue and spectator lists
- solo and 1v1 match stubs
- server-generated placeholder scores
- server-side ticket rewards
- first arcade token sticker placed in the first empty locker slot
- basic in-world placeholder cabinet and spectator spot parts

Out of scope:

- rhythm chart gameplay
- music/audio assets
- DDR input timing
- 2v2, FFA, co-op, or team modes
- final cabinet models, animation, VFX, or UI art
- economy or purchasable rewards

## Player Experience

1. Player walks to the placeholder arcade cabinet pair.
2. Player opens the arcade menu.
3. Player can join queue or watch.
4. First queued player can start Solo stub.
5. First two queued players can start 1v1 stub.
6. Active player can finish the stub match.
7. Server generates scores and awards tickets.
8. First successful arcade reward adds an `Arcade Token` decor item to the player's locker if an empty slot exists.

## Asset Dependency Rule

Use placeholders only:

- blockout cabinet parts
- colored pads/screens
- simple spectator spot markers
- text scoreboard
- basic Roblox UI controls

If final arcade models, songs, sound effects, rhythm charts, character animations, or UI art are needed, stop and tell the user exactly what is required.

## Acceptance Criteria

- `Workspace.World.Arcade.CabinetPair_001` exists with `PromptPart`, `ProximityPrompt`, pads, screens, and spectator spots.
- Arcade menu opens from code/UI and can show queue, spectators, active match, songs, and ticket count.
- Server owns queue/spectator/active match state.
- Server rejects match starts unless the player is near the cabinet and in the correct queue position.
- Solo and 1v1 stubs can start without rhythm charts.
- Stub finish produces server-generated scores.
- Rewards increment `arcadeTickets`.
- First arcade reward places `arcade_token_sticker` in the first empty locker slot when possible.
- No music, charts, or final assets are introduced.
