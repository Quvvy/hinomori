# Historical Implementation Plans

Files in this folder preserve how earlier Hinomori prototype slices were planned and verified. They are **not current requirements** and may describe superseded behavior, names, world paths, UI, limitations, or implementation status.

Fresh collaborators and coding agents should use:

- `../../AGENTS.md`
- `../CURRENT_STATE.md`
- `../GDD.md`
- `../ARCHITECTURE.md`
- `../STUDIO_WORLD.md`
- `../ROADMAP.md`
- current subsystem documents in `docs/`

Do not implement an archived plan without first reconciling it with current code and receiving a new approved slice.

Archived records include:

- numbered Slice 1-5 plans
- V0 hardening and note-owner-control plans
- superseded remaining-decisions record
- friend-note implementation plan now summarized in `SOCIAL_SYSTEMS.md`
- rhythm first-pass record now replaced by `RHYTHM_SYSTEM.md`
